# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

PrintWise is a Chrome Extension (Manifest V3) that uses the Claude Code CLI (via native messaging) to extract article content from web pages and render it with editorial-quality typography for reading and printing. No API key — uses your Claude subscription.

## Architecture

**No build system or package manager** — pure vanilla JS/HTML/CSS Chrome extension. Load as unpacked in Chrome.

### Data Flow

1. **popup.js** — On "Extract" click, injects `extractPageContent()` into the active tab via `chrome.scripting.executeScript`, then sends extracted text to the background worker via `chrome.runtime.sendMessage`.

2. **background.js** — Receives page text, forwards it to the native messaging host via `chrome.runtime.sendNativeMessage('com.printwise.native_host', ...)`.

3. **native-host/printwise_host.bat** — Windows batch file registered in the registry; Chrome spawns this. It calls `wsl.exe -d <distro> -u <user> -- python3 .../printwise_host.py`. Generated by `install.sh`.

4. **native-host/printwise_host.py** — Python script running inside WSL. Reads the native message, calls `claude -p <prompt>`, parses the JSON response, writes it back to stdout using the native messaging length-prefix protocol.

5. **popup.js** (continued) — Stores the result plus source URL in `chrome.storage.local` under `readerData`, opens reader.html in a new tab.

6. **reader.html / reader.js** — Reads `readerData` from storage and renders with editorial typesetting (Cormorant Garamond display, Source Serif 4 body, DM Sans UI). Print-optimised for A4.

### Setup (one-time)

```bash
cd native-host
./install.sh
```

The installer auto-detects your WSL distro, Linux username, and Windows username, then:
- Asks for the extension ID (from chrome://extensions)
- Writes `printwise_host.bat` and the manifest JSON to `%APPDATA%\Local\PrintWise\native-host\`
- Registers the host in the Windows registry under `HKCU\Software\Google\Chrome\NativeMessagingHosts\com.printwise.native_host`

After installing, reload the extension in Chrome.

### Debug log

The Python host logs to `/tmp/printwise-host.log` — check this first if extractions fail.

### Key Details

- This setup assumes Chrome runs on Windows and the Claude CLI runs in WSL. The `install.sh` bridges them via `wsl.exe`.
- Content extraction in `popup.js` uses site-specific strategies: Twitter/X tweet selectors, Substack `.body.markup`, generic article/main selectors, body fallback. Truncated to 12,000 chars.
- The Claude JSON response is parsed with a fallback that wraps raw text in `<p>` tags if JSON parsing fails.
- **printwise-preview.html** is a static demo page, not used by the extension at runtime.

### Permissions

- `activeTab`, `scripting`, `storage`, `nativeMessaging`
- CSP: `script-src 'self'; object-src 'self'`
- No host permissions — no outbound network from the extension
